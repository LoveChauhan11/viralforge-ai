# Project Context

Product: ViralForge AI / ForgeOS, a mobile-first production studio that turns creator-owned local media into high-retention YouTube Shorts.

MVP flow: onboard → upload/resume → media analysis → three AI directions → immutable EDL/version → storyboard edits → deterministic FFmpeg preview/master → explicit approval → YouTube upload → Option 1 mobile music handoff → analytics → conservative, reversible Creator DNA learning.

Fixed product rules:

- Output is 15–30 seconds; AI may choose duration; optional force-exactly-30 setting.
- Default publication requires creator/owner approval.
- YouTube Shorts-library music is never attached or rendered by ForgeOS. Recommend track/cue and guide the creator to add it in the YouTube mobile app.
- Original/licensed audio can be embedded with rights evidence.
- Biking/travel/action is the first optimized profile; architecture remains niche-neutral.
- Raw media is private, tenant-scoped, retention-managed, and never durable on Railway local disk.
- AI returns schema-valid decisions; deterministic code executes them.

Document precedence: security/privacy/rights → accepted ADRs/product decisions → product requirements → exact data/API/event/AI/media contracts → sprint backlog → implementation convenience.
